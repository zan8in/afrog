# 《Afrog 从参数到原理：-h 硬核拆解》系列总览

> “你以为你在调参数，其实你在调一套扫描系统。”

这一系列不走“手把手点按钮”的路线，专讲 **`afrog -h` 每一个参数** 背后的东西：

- 参数说明（从用户视角，到底在控制什么）
- 为什么要有这个参数（没有它会怎样，替代方案为何不够）
- 核心实现原理（从开发者视角：流程、关键模块、数据怎么流）
- 设计取舍（性能 / 稳定 / 准确 / 目标影响）
- 使用建议（保守 / 均衡 / 激进的配置思路）
- 实战打法（不同战场：SRC、红队、巡检、跑批告警、应急验证）

你会发现：很多参数不是“开关”，而是“战术选择”。

---

## 每期固定结构

- 这期讲哪些参数（先给结论：适用场景/一句话推荐）
- 参数说明（官方含义 + 真正会踩的坑）
- 为什么要有它（痛点与必要性）
- 核心实现原理（代码链路与关键模块）
- 设计取舍（为什么这么设计）
- 配置建议（3 套：保守 / 均衡 / 激进）
- 实战场景（怎么配、为什么这样配）
- 排障清单（现象 → 应对参数）
