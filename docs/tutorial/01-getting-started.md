## 🐸 Afrog 是个啥？

简单说，Afrog 是一款**高性能**、**高扩展性**的漏洞扫描工具。

为什么要用它？
- **快**：Go 语言开发，并发性能强劲。
- **准**：内置大量高质量 PoC（漏洞验证脚本），社区活跃更新快。
- **稳**：不仅能扫 Web 漏洞，还能联动端口扫描，甚至支持反连（OOB）检测。

它就像一只敏捷的青蛙，迅速捕捉网络中的“害虫”（漏洞）。

## 🛠️ 安装：三步上篮

不管你是 Windows、Mac 还是 Linux，安装 Afrog 都非常简单。我们推荐**直接下载二进制文件**，省去编译的烦恼。

### 方法一：直接下载（推荐 ⭐⭐⭐⭐⭐）

1.  访问 GitHub Releases 页面：[https://github.com/zan8in/afrog/releases](https://github.com/zan8in/afrog/releases)
2.  找到最新版本（Latest），根据你的系统下载对应的压缩包：
    -   **Windows**: `afrog_windows_amd64.zip`
    -   **Mac (Intel)**: `afrog_darwin_amd64.tar.gz`
    -   **Mac (M1/M2)**: `afrog_darwin_arm64.tar.gz`
    -   **Linux**: `afrog_linux_amd64.tar.gz`
3.  解压，把里面的 `afrog` (或者 `afrog.exe`) 放到你想放的地方。
4.  打开终端（CMD/PowerShell），进入该目录，运行：
    ```bash
    ./afrog -v
    ```
    如果看到了版本号，恭喜你，安装成功！

### 方法二：源码安装（适合 Go 语言玩家）

如果你本地有 Go 1.19+ 环境，一条命令搞定：

```bash
go install -v github.com/zan8in/afrog/cmd/afrog@latest
```

安装完成后，直接在终端输入 `afrog` 即可（前提是你的 GOPATH/bin 在环境变量里）。

## 🚀 Hello World：你的第一次扫描

安装好了，怎么用？最简单的用法只需要一个参数 `-t` (Target)。

假设我们要扫描一个测试站点 `http://example.com`（**请务必扫描已授权的目标！**）：

```bash
afrog -t http://example.com
```

### 你会看到什么？

终端会开始疯狂刷屏（别慌，这是它在干活）：
1.  **加载 PoC**：Afrog 会自动加载内置的漏洞规则。
2.  **开始扫描**：显示当前的进度。
3.  **发现漏洞**：
    -   如果看到 <font color="red">**[CRITICAL]**</font> 或 <font color="red">**[HIGH]**</font>，说明发现了高危漏洞！
    -   终端会显示漏洞名称、URL 和 PoC ID。

### 报告在哪里？

扫描结束后，Afrog 贴心地为你生成了一份 **HTML 报告**。
默认情况下，报告会保存在当前目录的 `reports` 文件夹下，文件名是日期时间，比如 `20230101-120000.html`。

双击打开它，你会看到一个精美的网页，里面详细列出了：
-   扫描目标
-   发现的漏洞列表
-   每个漏洞的请求（Request）和响应（Response）包 —— **这对复现漏洞至关重要！**

## 💡 小贴士

1.  **第一次运行慢？**
    Afrog 第一次运行时会自动下载最新的 PoC 规则库，取决于网速，可能会稍慢一点点，耐心等待即可。
2.  **不仅是 URL**
    `-t` 参数不仅支持 `http://...`，也支持 IP 地址（如 `192.168.1.1`）。
